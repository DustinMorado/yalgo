Parser.template_arg
Argument template for Parser. This is a sample argument object for passing into add_argument.

l_opt and s_opt flags cannot be used for positional arguments, but at least one must be given for an option argument.

Option arguments cannot be required and not take arguments.

Positional arguments cannot take arguments themselves.

Required arguments cannot specify a default value.

Description and meta_values are not mandatory unless you use the display_help function (But of course you're providing adequate documentation right? ;))

Fields:

  • name Argument name used as field in returned option table.
  • l_opt Specifies option long style flag
  • s_opt Specifies option short style flag
  • is_pos Indicaties whether argument is positional or an option
  • has_arg Indicates whether option takes an argument
  • is_reqd Indicates whether option is mandatory or not
  • dflt_val Specifies a default value to be used for the option
  • descr Option description to be used for help
  • meta_val Specifies argument placeholder string used in help
Parser:add_arg (_arg)
Add an argument to a parser object This function lets the parser know what arguments to look for at the command line.

Parameters:

  • _arg An argument table with the fields as described in template_arg

Usage:

    my_parser:add_arg({name = 'option', l_opt = '--opt', ... })

Usage:

Parser:get_args (args)
Get arguments takes an arg table and updates the parser arg table values returning a table indexed by argument name storing the found and default values.

Parameters:

  • args Sequence table of arguments, default is _G.arg

Returns:

    A table indexed by parser argument name with found and default values

Usage:

    found_args = my_parser:get_args()
